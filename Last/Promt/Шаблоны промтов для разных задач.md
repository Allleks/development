## **Для обучения:**
```
Я изучаю [тема] на уровне [новичок/продвинутый].
Объясни [концепт] как будто мне [10/20/30] лет.
Используй аналогии из [предметная область].
Приведи пример на [язык программирования].
Задай мне 3 контрольных вопроса для проверки понимания.
```
### **Для кодирования:**
```
Напиши класс [ИмяКласса] на C++/Qt, который:
1. Наследуется от [БазовыйКласс]
2. Имеет методы: [метод1, метод2]
3. Использует [паттерн/принцип]
4. Обрабатывает ошибки через [механизм]
5. Оптимизирован для [критерий]

Предоставь:
- Заголовочный файл с документацией
- Файл реализации с комментариями
- Unit-тесты для ключевых методов
- Пример использования в 3 строках
```
### **Для отладки:**
```
У меня ошибка в коде на Qt:
[вставь код]

Ошибка компиляции/рантайма:
[текст ошибки]

Я уже попробовал:
1. [попытка 1]
2. [попытка 2]

Проанализируй:
1. В чем причина ошибки
2. Как исправить с объяснением
3. Как предотвратить подобное в будущем
4. Альтернативные реализации
```
### **Для оптимизации:**
```
Проанализируй этот код на предмет оптимизации:
[вставь код]

Контекст:
- Размер данных: [объем]
- Частота вызовов: [частота]
- Целевая платформа: [платформа]

Предложи:
1. 3 немедленных улучшения производительности
2. Архитектурные изменения на будущее
3. Инструменты для профилирования
4. Метрики для измерения улучшений
```

## Техники улучшения ответов

### **Уточняющие вопросы:**
Если ответ неполный, спроси:
```
Уточни [конкретный аспект]
Приведи пример для [конкретный случай]
Сравни с [альтернативное решение]
Объясни как это работает при [условие]
```
### **Итеративное уточнение:**
1. Получи общий ответ
2. Уточни детали:
```
Отлично! Теперь:
1. Добавь обработку ошибок для метода addItem()
2. Оптимизируй поиск по индексу
3. Добавь поддержку undo/redo
4. Напиши unit-тест для граничных случаев
```
### **Запрос примеров:**
```
Не только объясни концепцию, но и покажи:
1. Минимальный рабочий пример
2. Пример из реального проекта
3. Пример неправильного использования
4. Пример с лучшими практиками
```
## 6. Промты для глубокого понимания:
```
Объясни [концепт] на 3 уровнях:
1. Для начинающего (простая аналогия)
2. Для практикующего (как использовать)
3. Для эксперта (внутреннее устройство)

Используй:
- Код на C++/Qt
- Диаграммы в ASCII
- Сравнение с альтернативами
- Ссылки на документацию
```
