___
Ключевые выводы
1. **Модель ≠ Данные** - модель это интерфейс к данным
2. **Всегда эмитить сигналы** при изменении данных
3. **Использовать правильные begin/end пары**
4. **Оптимизировать для больших данных** (lazy loading, кеш)
5. **Proxy-модели** для сортировки/фильтрации
6. **[[Создание и использование индексов|Проверять индексы]] на валидность
7. **Разделять ответственность** - модель только данные, представление только UI
---
Чек-лист
```cpp
// При создании модели:
[ ] Наследоваться от QAbstractItemModel/TableModel/ListModel
[ ] Реализовать обязательные методы
[ ] Добавить сигналы для изменений
[ ] Реализовать проверку индексов

// При изменении данных:
[ ] emit beginInsertRows/beginRemoveRows
[ ] Изменить данные
[ ] emit endInsertRows/endRemoveRows

// При обновлении данных:
[ ] emit dataChanged

// При полной перезагрузке:
[ ] emit beginResetModel
[ ] Загрузить новые данные
[ ] emit endResetModel
```
---
**Золотое правило:** Модель должна быть "тупой" - только хранение и доступ к данным. Вся бизнес-логика в отдельном слое.
**Правило:** Если меняется внешний вид - делегат. Если меняются данные - модель.
