## **ЧЕК-ЛИСТ ДЛЯ КАЖДОГО КЛАССА:**
1. **S** - Класс делает одну вещь?
2. **O** - Можно расширить без изменения кода?
3. **L** - Можно подставить наследника без поломки?
4. **I** - Интерфейс минимальный и специфичный?
5. **D** - Зависит от абстракций, а не реализаций?

## **БЫСТРЫЕ ПРИЗНАКИ НАРУШЕНИЯ SOLID:**
- Класс > 500 строк → **S**
- Много `if/switch` по типам → **O**
- Методы `throw "Not implemented"` → **L, I**
- `#include` конкретных классов → **D**
- Модификация при добавлении фичи → **O**

**Запомните:** SOLID — это не догма, а ориентир. В Qt иногда проще нарушить принцип, чем усложнять код. Главное — понимать последствия.
