- `ID` - это **первичный ключ (PRIMARY KEY)**
- Он **уникально идентифицирует** каждую запись
- `IDENTITY(1,1)` означает, что он **автоматически проставляется** (начиная с 1, увеличиваясь на 1)

- `devID` - это **внешний ключ (FOREIGN KEY)**
- Он связывает текущую таблицу с таблицей `Общие параметр`
```text
┌────────────────────────────────────────────────────┐
│  Таблица Проемы                                    │
├──────┬─────────┬───────────────────────────────────┤
│  ID  │ devID   │ Другие поля                       │
├──────┼─────────┼───────────────────────────────────┤
│  1   │ 5       │ ...                               │
│  2   │ 5       │ ... ← один devID может быть       │
│  3   │ 7       │ ...   в нескольких записях        │
│  4   │ 5       │ ...                               │
└──────┴─────────┴───────────────────────────────────┘
  ↑                     ↑
  PRIMARY KEY          FOREIGN KEY
  (уникален)           (может повторяться)
```
___
**CONSTRAINT** (ограничение) - это правило, которое защищает данные от ошибок.
```text
┌────────────────────────────────────────────────────┐
│  CONSTRAINT = ЗАЩИТНЫЙ БАРЬЕР ДЛЯ ДАННЫХ           │
├────────────────────────────────────────────────────┤
│  Виды ограничений:                                 │
│  ┌─────────────────────────────────────────────┐   │
│  │ PRIMARY KEY   - защита от дубликатов        │   │
│  │ FOREIGN KEY   - защита от битых ссылок      │   │
│  │ UNIQUE        - защита от повторов          │   │
│  │ CHECK         - защита от неправильных чисел│   │
│  │ DEFAULT       - защита от пустоты           │   │
│  └─────────────────────────────────────────────┘   │
└────────────────────────────────────────────────────┘
```
