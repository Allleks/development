```sql
USE [Doodle_x_Angarsk];
GO
ALTER TABLE [Проемы]
ADD [Расположение] nvarchar(MAX) COLLATE Cyrillic_General_CI_AS NULL;
GO

USE [Doodle_x_Angarsk];
GO
ALTER TABLE [Проемы]
ADD [Масса] decimal(9,3) NULL;
GO
```
### Пояснение:
- **`ALTER TABLE [Проемы]`** - изменяем таблицу Проемы
- **`ADD [Расположение]`** - добавляем новую колонку с таким названием
- **`nvarchar(MAX)`** - тип данных (строка любой длины)
- **`NULL`** - поле может быть пустым (необязательное)
- **`GO`** Это разделитель пакетов, который понимают только инструменты (SSMS, Azure Data Studio, dbviewr).

Если хочешь добавить поле с каким-то значением по умолчанию:
```sql
USE [Doodle_x_Angarsk];
GO
-- С пустой строкой по умолчанию
ALTER TABLE [Проемы]
ADD [Расположение] nvarchar(MAX) COLLATE Cyrillic_General_CI_AS NOT NULL
CONSTRAINT DF_Проемы_Расположение DEFAULT '';
GO
-- При вставке без указания: = '' (пустая строка)

-- Вариант: Со значением по умолчанию
ALTER TABLE [Проемы]
ADD [Расположение] nvarchar(MAX) NOT NULL
CONSTRAINT DF_Проемы_Расположение DEFAULT 'не указано';
-- При вставке без указания: = 'не указано'
```
### Проверка после добавления:
```sql
-- Посмотреть структуру таблицы с новым полем
SELECT * FROM [Проемы];
-- или
SELECT COLUMN_NAME, DATA_TYPE 
FROM INFORMATION_SCHEMA.COLUMNS 
WHERE TABLE_NAME = 'Проемы'
ORDER BY ORDINAL_POSITION;
```
После добавления поля можешь вставлять в него данные:
```sql
INSERT INTO [Проемы] (devID, [Расположение])
VALUES (1, 'какое-то значение');
```

## Типичные команды ALTER TABLE:
```sql
-- 1. Добавить колонку
ALTER TABLE Таблица ADD Колонка тип NULL;
-- 2. Удалить колонку
ALTER TABLE Таблица DROP COLUMN Колонка;
-- 3. Изменить тип данных
ALTER TABLE Таблица ALTER COLUMN Колонка новый_тип;
-- 4. Добавить ограничение (constraint)
ALTER TABLE Таблица 
ADD CONSTRAINT CK_Колонка CHECK (Колонка > 0);
-- 5. Добавить значение по умолчанию
ALTER TABLE Таблица 
ADD CONSTRAINT DF_Колонка DEFAULT 0 FOR Колонка;
-- 6. Переименовать колонку
EXEC sp_rename 'Таблица.СтароеИмя', 'НовоеИмя', 'COLUMN';
```

Про **[]** в DDL
**Квадратные скобки** — это **способ экранирования идентификаторов** (имен таблиц, колонок, constraint'ов) в SQL Server.
1. **Не содержит пробелов** (в отличие от `[Место расположения проемов]`)
2. **Не содержит спецсимволов** (дефисов, точек и т.д.)
3. **Не является зарезервированным словом** SQL
